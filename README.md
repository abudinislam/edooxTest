# Edoox — Тренажёр для подготовки к экзаменам

Android-приложение для подготовки к ЕНТ. Ученик выбирает предмет, проходит тест из 5 заданий разных типов, затем видит разбор с объяснениями.

## Начало работы

### 1. Получите Gemini API Key

1. Перейдите на [Google AI Studio](https://aistudio.google.com)
2. Создайте API Key (бесплатно)
3. Скопируйте `local.properties.example` в `local.properties`
4. Вставьте ключ в `local.properties`:
   ```
   GEMINI_API_KEY=ваш_ключ_здесь
   ```

Если `local.properties` отсутствует или ключ не указан, приложение запустится, но генерация заданий и проверка открытых ответов через Gemini работать не будут.

### 2. Сборка и запуск

```bash
./gradlew assembleDebug
```

## Архитектура

```
app/src/main/java/abubakir/edooxtest/
├── data/
│   ├── model/          # Question (sealed), UserAnswer (sealed), QuestionResult
│   ├── remote/         # Retrofit интерфейс + DTO для Gemini API
│   └── repository/     # QuizRepository — генерация и проверка через LLM
└── ui/
    ├── MainActivity    # Точка входа
    ├── EdooxApp        # Compose navigation graph
    ├── subject/        # SubjectScreen + SubjectViewModel
    ├── test/           # TestScreen + TestViewModel
    ├── result/         # ResultScreen + ResultViewModel
    └── theme/          # ColorScheme / Typography / Shapes
```

Приложение построено по MVVM. UI работает через `ViewModel`, сетевые вызовы и обработка ответов вынесены в `Repository`.

## Схема JSON для заданий

```json
{
  "questions": [
    {
      "id": "1",
      "type": "multiple_choice",
      "text": "Чему равно производное sin(x)?",
      "options": ["cos(x)", "-cos(x)", "sin(x)", "-sin(x)"],
      "correct_answer": "cos(x)",
      "explanation": "Производная sin(x) = cos(x) по таблице производных."
    },
    {
      "id": "2",
      "type": "open_ended",
      "text": "Объясните теорему Пифагора.",
      "sample_answer": "В прямоугольном треугольнике квадрат гипотенузы равен сумме квадратов катетов: a²+b²=c².",
      "explanation": "Ответ должен содержать формулу и условие применимости (прямоугольный треугольник)."
    },
    {
      "id": "3",
      "type": "matching",
      "text": "Сопоставьте понятия с определениями",
      "pairs": [
        {"left": "Интеграл", "right": "Площадь под кривой"},
        {"left": "Производная", "right": "Скорость изменения функции"},
        {"left": "Предел", "right": "Значение, к которому стремится функция"},
        {"left": "Логарифм", "right": "Показатель степени"}
      ],
      "explanation": "Эти понятия — основы математического анализа."
    }
  ]
}
```

**Состав теста**: 2 закрытых + 2 открытых + 1 на соответствие (фиксированный набор типов в промпте).

## Как устроена проверка трёх типов

| Тип | Где проверяется | Логика |
|-----|-----------------|--------|
| **Закрытый** (multiple_choice) | Клиент | Сравнение `userAnswer.selected == question.correctAnswer` |
| **На соответствие** (matching) | Клиент | Сравнение `Map<String,String>` ответов с правильными парами |
| **Открытый** (open_ended) | LLM (второй запрос) | Gemini возвращает `{"is_correct": bool, "feedback": "..."}` |

Открытые ответы проверяются после нажатия «Завершить» — один API-вызов на каждый открытый вопрос (последовательно, с `suspend`).

## Обработка ошибок

- Сетевые ошибки при генерации показываются на экране выбора предмета, пользователь остаётся в текущем состоянии.
- Ошибка парсинга JSON обрабатывается через `try/catch`; если Gemini вернёт JSON в markdown-обёртке, она удаляется перед парсингом.
- Ошибка проверки открытого ответа не прерывает тест: такой ответ помечается как неверный с пояснением в фидбеке.
- Для пустого ключа и проблем квоты Gemini возвращаются отдельные сообщения.
- Генерацию можно отменить во время загрузки.

## Решения и компромиссы

1. **Сериализация вопросов**: вместо `Parcelize` используется ручная десериализация через Gson + `type`-поле.

2. **Matching без автозаполнения**: вопрос на соответствие начинается с пустых значений. Кнопка «Далее» активируется только после заполнения всех соответствий.

3. **Без DI-фреймворка**: для такого объёма `QuizRepository` создаётся прямо во `ViewModel`.

4. **Одна Activity + Compose Navigation**: навигация между экранами сделана внутри одного `MainActivity`.

5. **Gemini Flash-Lite**: выбран `gemini-2.5-flash-lite` для структурированного JSON-ответа.
